# EA MQL5 Fix Playbook - Production Ready Guide
**Version**: 2.1
**Target**: MQL5 Build 4170+ (MetaTrader 5)
**Updated**: 2025-08-16

## Purpose & Objectives
- Achieve **clean compile** (0 errors, 0 warnings) for all EA components
- Apply modern **MQL5 standards** (OOP, handle lifecycle, structured logging, error recovery)
- Ensure **production stability** with proper initialization, synchronization, and performance optimization
- Provide **evidence-based fixes** for common compilation and runtime issues
- **CRITICAL**: Detect and eliminate **hidden errors** that bypass automated compilation logs

## ⚠️ HIDDEN ERROR DETECTION PROTOCOL
**Problem**: Some compilation errors are not captured by automated scripts but appear in direct MetaEditor compilation.

**Root Cause**: Compilation log parsing may miss certain error patterns or timing-sensitive issues.

**Detection Strategy**:
1. **Always perform manual verification** after automated "success"
2. **Compile directly in MetaEditor** as final validation step
3. **Check for missing function implementations** even if declarations exist
4. **Verify const correctness** across all class methods

## 🚨 HIDDEN ERROR PATTERNS - CRITICAL KNOWLEDGE

### Pattern 1: Missing Function Implementation
**Symptom**: `function 'ClassName::FunctionName' must have a body`
**Root Cause**: Function declared in class but implementation missing
**Detection**: Only appears in direct MetaEditor compilation, NOT in automated logs
**Fix Strategy**:
```cpp
// ❌ Declaration only (causes hidden error)
class CLogger {
    void Log(const string message) const;
};

// ✅ Add implementation
void CLogger::Log(const string message) const {
    // Implementation here
}
```

### Pattern 2: Const Correctness Violation
**Symptom**: `call non-const method for constant object`
**Root Cause**: const method trying to modify object state
**Detection**: May not appear in all compilation runs
**Fix Strategy**:
```cpp
// ❌ const method modifying state
void Log(const string msg) const {
    m_queue.Add(msg); // Modifies state!
}

// ✅ Remove const or make state mutable
void Log(const string msg) {
    m_queue.Add(msg); // OK now
}
```

### Pattern 3: Struct Assignment Deprecation
**Symptom**: `initialized from type using assignment operator, deprecated`
**Root Cause**: Direct struct assignment from function return
**Fix Strategy**:
```cpp
// ❌ Deprecated pattern
StructType var = FunctionReturningStruct();

// ✅ Modern MQL5 pattern
StructType var;
var = FunctionReturningStruct();
```

### Chiến lược sửa lỗi theo chuẩn MQL5
- Truy cập object
  - MQL5 không dùng `->` cho object. Luôn dùng dấu `.` kể cả khi biến là con trỏ object.
  - Dấu hiệu: `error 223 '>' operand expected`, `error 237 object pointer expected` → thay `obj->Method()` bằng `obj.Method()`.
- const-correctness
  - Tránh `const_cast`. Nếu cần gọi từ context const, hãy khai báo method là `const` nếu không thay đổi trạng thái.
  - Ví dụ: `double GetBollingerBands(...) const` để cho phép gọi trong phân tích chỉ đọc.
- Indicator handles & CopyBuffer
  - Tạo handle, lấy dữ liệu bằng `CopyBuffer`, giải phóng bằng `IndicatorRelease`.
  - Tránh dùng hàm trả trực tiếp giá trị double kiểu MQL4-style.
  - Đảm bảo mảng đích đủ kích thước, kiểm tra giá trị trả về của `CopyBuffer` (>0).
- Khởi tạo biến
  - Luôn khởi tạo trước khi dùng: `double upper=0.0, middle=0.0, lower=0.0;`
  - Tránh cảnh báo “possible use of uninitialized variable”.
- Ép kiểu an toàn
  - Volume từ series là `long`. Khi tính toán với `double`, ép kiểu rõ ràng: `(double)iVolume(...);`
  - Spread: dùng `SymbolInfoInteger(symbol, SYMBOL_SPREAD)` → ép `long→double` và nhân `_Point` nếu cần điểm/point.
- Khối lượng giao dịch (volume digits)
  - Không phụ thuộc `SYMBOL_VOLUME_DIGITS`. Tính từ bước khối lượng:  
    `volStep = SymbolInfoDouble(symbol, SYMBOL_VOLUME_STEP);`  
    `volDigits = (volStep>0) ? (int)MathRound(-MathLog10(volStep)) : 2;` (giới hạn [0..8])
- Struct reset
  - Không dùng `memset` cho struct. Viết hàm `Reset...()` để gán từng field về giá trị mặc định an toàn.
- IsTradeAllowed & môi trường
  - Tránh `IsTradeAllowed()` tùy nền tảng. Dùng:  
    `TerminalInfoInteger(TERMINAL_TRADE_ALLOWED) != 0 && AccountInfoInteger(ACCOUNT_TRADE_ALLOWED) != 0`.
- Enum & switch-case
  - Tránh trùng `case` giá trị; hợp nhất hoặc loại bỏ case dư thừa.
- Tham chiếu (&) với kiểu không phù hợp
  - Khi gặp `error 229 reference cannot used`, cân nhắc trả về “theo giá trị” thay vì `const &` cho struct nhỏ/trung bình.
- Khởi tạo hệ thống (OnInit) không-fatal
  - Các phân tích yêu cầu “đồng bộ timeframe/cache” có thể trả false ở tick đầu. Đặt `m_initialized = true` trước khi gọi phân tích đầu, và nếu `Update...()` thất bại do chờ đồng bộ thì ghi log “deferred” và tiếp tục. Tránh `return INIT_FAILED` chỉ vì chưa có nến/tick mới.
- Vòng đời (Lifecycle) & dọn dẹp
  - Trong `OnDeinit`: gọi `Deinitialize()` nếu có, `delete` object, gán `NULL` để an toàn.

- Deferred initialization pattern (tránh INIT_FAILED do đồng bộ)
```cpp
class CEACore {
private:
    bool m_initialized;
    bool m_deferredInitComplete;
    datetime m_lastInitAttempt;
    int m_initAttempts;
    
public:
    CEACore() : m_initialized(false), m_deferredInitComplete(false), 
                m_lastInitAttempt(0), m_initAttempts(0) {}
    
    bool Initialize() {
        m_initialized = true; // Mark as initialized to allow OnTick
        return true; // Don't fail OnInit
    }
    
    void OnTick() {
        if(!m_deferredInitComplete) {
            if(!AttemptDeferredInit()) {
                return; // Skip this tick, try again later
            }
        }
        
        // Normal tick processing
        ProcessTick();
    }
    
private:
    bool AttemptDeferredInit() {
        datetime currentTime = TimeCurrent();
        
        // Throttle init attempts (max once per 5 seconds)
        if(currentTime - m_lastInitAttempt < 5) {
            return false;
        }
        
        m_lastInitAttempt = currentTime;
        m_initAttempts++;
        
        // Try to initialize indicators/data that might not be ready
        if(!InitializeIndicators()) {
            Print("[DEFERRED] Waiting for market data... attempt ", m_initAttempts);
            if(m_initAttempts > 20) { // Max 20 attempts
                Print("[ERROR] Deferred init failed after 20 attempts");
                return false;
            }
            return false;
        }
        
        if(!InitializeAnalysis()) {
            Print("[DEFERRED] Waiting for analysis modules... attempt ", m_initAttempts);
            return false;
        }
        
        m_deferredInitComplete = true;
        Print("[SUCCESS] Deferred initialization completed after ", m_initAttempts, " attempts");
        return true;
    }
    
    bool InitializeIndicators() {
        // Check if we have enough bars
        if(iBars(_Symbol, PERIOD_CURRENT) < 100) {
            return false;
        }
        
        // Try to create indicators
        int emaHandle = iMA(_Symbol, PERIOD_CURRENT, 20, 0, MODE_EMA, PRICE_CLOSE);
        if(emaHandle == INVALID_HANDLE) {
            return false;
        }
        
        // Test if we can get data
        double emaBuffer[1];
        if(CopyBuffer(emaHandle, 0, 0, 1, emaBuffer) <= 0) {
            IndicatorRelease(emaHandle);
            return false;
        }
        
        IndicatorRelease(emaHandle); // Cleanup test handle
        return true;
    }
    
    bool InitializeAnalysis() {
        // Initialize analysis modules that depend on market data
        // Return false if not ready, true if successful
        return true;
    }
    
    void ProcessTick() {
        // Your main EA logic here
        Print("Processing tick - EA fully initialized");
    }
};

// Usage in main EA
CEACore* g_eaCore = NULL;

int OnInit() {
    Print("=== EA INITIALIZATION START ===");
    
    g_eaCore = new CEACore();
    if(!g_eaCore || !g_eaCore.Initialize()) {
        Print("❌ Failed to create EA core");
        return INIT_FAILED;
    }
    
    Print("✅ EA initialized - deferred init will complete on first ticks");
    return INIT_SUCCEEDED; // Always succeed, defer complex init
}

void OnTick() {
    if(g_eaCore) {
        g_eaCore.OnTick();
    }
}

void OnDeinit(const int reason) {
    if(g_eaCore) {
        delete g_eaCore;
        g_eaCore = NULL;
    }
}
```

### Bảng tra nhanh lỗi thường gặp → Cách sửa
- **223/237/157/183 (pointer/object, dấu ngoặc)**: thay `->` bằng `.`, soát cặp `()` và `if/else` cân bằng.
- **256 undeclared identifier**: kiểm tra include, phạm vi, tên hàm đúng chuẩn MQL5, tránh API cũ.
- **199 wrong parameters count**: đối chiếu chữ ký hàm MQL5 (đặc biệt indicator constructors).
- **229 reference cannot used**: đổi sang return-by-value hoặc bỏ `&` với kiểu tạm/không hợp lệ.
- **150 illegal operation use**: biểu thức logic trên kiểu không hợp lệ (vd: `if(ptr && logger)` → thay `ptr!=NULL`).
- **warning 43 long→double**: ép kiểu `(double)` trước khi tính toán.

### Logging & Error Handling
- Tối thiểu cần logger ghi file và console; phân cấp mức LOG (`DEBUG/INFO/WARN/ERROR`).
- Với chức năng chưa sẵn sàng (Telegram, metrics OS), cài “stub” an toàn (ghi log, trả `false`).
- Ghi rõ lý do defer/skip phân tích (chờ đồng bộ timeframe, cache chưa sẵn sàng).

### Hiệu năng & Đồng bộ
- Cache dữ liệu indicator; chỉ cập nhật khi có nến mới (`iTime(...)` đổi) hoặc timeout cache.
- Tránh gọi tạo handle indicator trên mỗi tick; tạo một lần, tái sử dụng; hoặc dùng lớp quản lý indicator.
- `ArraySetAsSeries` phù hợp hướng dữ liệu; chú ý chỉ số [0] là thanh mới nhất.

### Chuẩn hoá kiến trúc & đặt tên
- Tên hàm phản ánh hành động (verb), biến phản ánh ý nghĩa (noun).
- Tách lớp: Orchestrator/Manager/Analyzer theo trách nhiệm; không để “God class”.
- API công khai (public) tối thiểu cần thiết; còn lại private/protected.

### Checklist trước khi bàn giao (Release Gate)
- [ ] Compile toàn bộ: 0 errors, 0 warnings.
- [ ] Không dùng `->`, không `const_cast`, không `memset` struct.
- [ ] Indicator: tạo handle → CopyBuffer → Release; kiểm tra giá trị trả về.
- [ ] Ép kiểu rõ ràng `long→double`; biến được khởi tạo mặc định an toàn.
- [ ] Không phụ thuộc `SYMBOL_VOLUME_DIGITS`; tính từ `SYMBOL_VOLUME_STEP`.
- [ ] `OnInit` không fail vì đồng bộ; deferred init/log OK.
- [ ] `OnDeinit` giải phóng tài nguyên đầy đủ, gán `NULL`.
- [ ] Chạy Strategy Tester 1-2 tháng dữ liệu: OnInit OK, tick đầu tiên OK, không spam log, không chậm bất thường.
- [ ] Log framework hoạt động, mức WARN/ERROR rõ ràng.
